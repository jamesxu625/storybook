# L1 标准模板 — 产品需求文档（PRD）

> 绘本乐高 · 3-5 岁 · "按图搭"
> 版本：v0.1 · 2025-06-25

---

## 一、产品概述

### 目标用户
- **使用者**：3-5 岁儿童（不认字或初识字，能点选和涂色，操作能力有限）
- **付费方**：父母（关注孩子创造力/英语启蒙/成长记录）

### 核心体验
孩子通过 **选角色 → 选场景 → 屏幕涂色 → 听故事** 的极简流程，10 分钟"搭"出一本属于自己的有声双语互动绘本。全程不需要打字、不需要阅读复杂文字，靠图标和语音引导。

### 设计原则
1. **零文字依赖**：所有操作靠图标/图片/语音引导，4 岁孩子能独立走完
2. **每步有即时反馈**：点选有动画、涂色有声音、完成有庆祝
3. **最多 3 步选择**：每页决策不超过 3 个选项，避免选择困难
4. **大人可离开**：流程设计为孩子能自主完成，父母只需在开始时帮选故事主题

## 二、核心用户流程

### 主流程（做一本绘本）

```
[首页] 选故事主题
  ↓
[选角色] 从 3 个角色中选 1 个 + 起名字（语音/预设）
  ↓
[逐页搭建] 重复 N 页（默认 6 页）:
  ├─ [选场景] AI 给 2-3 个场景图，孩子点选
  ├─ [涂色] 在线稿上涂色（手指触控填色）
  └─ [听这页] AI 自动生成配音，播放这一页的故事
  ↓
[完成] 生成完整绘本
  ├─ 自动播放整本（翻页+配音）
  ├─ "给爸爸妈妈看！" → 生成分享二维码
  └─ "再做一本！" → 回首页
```

### 流程时长目标
- 单页搭建：1-2 分钟（选场景 30s + 涂色 60s + 听 20s）
- 完整一本：8-12 分钟（6 页）
- 孩子注意力能覆盖的范围内完成

## 三、页面与交互设计

### 3.1 首页 — 选故事主题

**目的**：孩子选择今天要搭什么故事

**界面**：
- 顶部：大标题"今天搭什么故事？"（带语音朗读）
- 主体：3-4 个主题卡片，每个是一张大图 + 图标
  - 🦕 小恐龙去冒险
  - 🚗 小汽车旅行记
  - 🏰 公主与城堡
  - 🐠 海底好朋友
- 每张卡片点击有放大动画 + 音效

**交互**：
- 点击主题卡片 → 进入选角色
- 卡片自动朗读主题名（语音引导）

### 3.2 选角色 — 选主角

**目的**：孩子选择故事主角并起名

**界面**：
- 顶部：选好的主题 + "选一个好朋友！"
- 主体：3 个角色立绘（同一画风，不同造型）
  - 比如恐龙主题：绿色恐龙、粉色恐龙、蓝色恐龙
- 角色名输入：预设 3 个名字选项（豆豆/奇奇/乐乐），点击选择，不需要打字

**交互**：
- 点击角色 → 角色跳一下 + "你好！我是XX！"（语音）
- 点名字选项 → 确认名字
- 底部大按钮"开始搭故事！" → 进入逐页搭建

### 3.3 逐页搭建 — 核心循环（重复 6 页）

这是产品的心脏。每页三个子步骤：

#### 3.3a 选场景

**界面**：
- 顶部："第 X 页 · XX 去哪里？"（语音朗读）
- 主体：2-3 个场景缩略图（AI 预生成的黑白线稿）
  - 比如恐龙主题第 2 页：森林、河边、山洞
- 每个场景有图标提示（🌲 🏞️ 🏔️）

**交互**：
- 点击场景 → 场景放大 + 音效"好选择！"
- 自动进入涂色

#### 3.3b 涂色

**界面**：
- 全屏显示选中的场景线稿（黑白轮廓）
- 底部色板：6-8 种颜色圆点（红/橙/黄/绿/蓝/紫/粉/棕）
- 当前选中的颜色有高亮

**交互**：
- 点颜色 → 该颜色高亮 + 语音"红色！"
- 手指点击/拖动线稿区域 → 填充该区域颜色（flood fill 模式，点一个封闭区域填满）
- 涂错可以点"↩️"撤销
- 底部"✅ 完成涂色"按钮 → 进入听故事

**技术要点**：
- Canvas + flood fill 算法实现区域填色
- 线稿需要预处理：封闭区域检测
- 触控优化：手指粗区域映射到精确点击点

#### 3.3c 听这页

**界面**：
- 显示孩子涂好色的画面（带角色）
- 底部文字条：中文故事 + 英文翻译（小字）
- 播放按钮：🔊

**交互**：
- 自动播放中文配音（edge-tts 生成）
- 可点"🔊 English"听英文
- 底部"下一页 →" → 回到选场景（下一页）
- 最后一页完成 → 进入完成页

### 3.4 完成页 — 作品发布

**目的**：给孩子成就感 + 让父母看到

**界面**：
- 全屏庆祝动画（彩带/星星）
- "XX 的《小恐龙去冒险》完成啦！"（语音朗读）
- 绘本封面预览
- 三个大按钮：
  - ▶️ **看我的书**（自动播放整本，翻页+配音）
  - 📱 **给爸爸妈妈看**（生成二维码，扫码可看）
  - 🆕 **再做一本**（回首页）

**交互**：
- 自动播放庆祝音效
- "看我的书"：进入阅读模式（复用现有导出书的播放器）
- "给爸爸妈妈看"：显示二维码 + 分享链接

## 四、AI 的角色（技术设计）

L1 中 AI 做三件事，全部在后端完成，孩子无感知：

### 4.1 场景线稿生成
- 输入：主题 + 页码 + 上一页情节
- 输出：2-3 个场景的黑白线稿（适合涂色）
- 技术：AI 生图 + 后处理转线稿（边缘检测/风格化）
- 缓存：同一主题+页码的场景可复用

### 4.2 故事文本生成
- 输入：主题 + 角色 + 孩子选的场景序列
- 输出：每页中英文故事文本（1-2 句，简单）
- 技术：GLM 生成，年龄适配（3-5 岁词汇）
- 时机：孩子选完全部场景后一次性生成（或每页选完即时生成）

### 4.3 配音生成
- 输入：故事文本
- 输出：中英文 MP3
- 技术：edge-tts（复用现有 tts.py）
- 时机：文本生成后自动配音

### AI 交互原则
- **AI 给选项，孩子做选择** —— AI 不直接决定内容
- **孩子感知不到 AI** —— 对孩子来说就是"魔法"自动生成
- **父母能看到 AI 做了什么** —— 成品里标注哪些是 AI 生成的

## 六、技术方案

### 前端（PWA）
- 纯 HTML/CSS/JS，无框架（与现有工具链一致）
- Canvas API 实现涂色（flood fill）
- Web Speech API / Audio 播放配音
- Service Worker 实现离线 PWA
- 响应式：手机/平板优先

### 后端
- Flask（复用现有 app.py 架构）
- 新增 API：
  - `GET /api/themes` — 获取主题/角色/场景库
  - `POST /api/story` — 根据选择生成故事文本 + 配音
  - `POST /api/lineart` — 生成场景线稿（或返回预制的）
- 复用：edge-tts 配音、导出 HTML

### 涂色技术
- 线稿图加载到 Canvas
- flood fill 算法（点击像素 → 填充连通区域）
- 颜色替换而非叠加（适合儿童，不需要混色）
- 保存为 PNG data URL

## 七、L1 MVP 范围

### MVP 做（P0）
- [ ] 1 个主题（恐龙）完整流程跑通
- [ ] 2 个角色可选
- [ ] 4 个场景可选
- [ ] 屏幕涂色（6 色板 + flood fill）
- [ ] AI 生成故事文本（中英）
- [ ] edge-tts 配音
- [ ] 完成后自动播放
- [ ] 导出/分享

### MVP 不做（留后续）
- 多主题（先验证 1 个）
- 角色命名（用预设名）
- 互动游戏（L1 先不加，专注搭+涂+听）
- 作品集（先做单本）
- 账户系统
- 付费墙

## 八、成功指标（阶段1验证）

| 指标 | 目标 | 含义 |
|------|------|------|
| 完成率 | >70% 开始做的孩子能完成一本 | 流程不卡、不挫败 |
| 次日回访 | >30% 孩子第二天主动想做 | 留存假设初步成立 |
| 分享率 | >50% 完成的绘本被分享给家人 | 成就感 + 传播 |
| 父母付费意愿 | >20% 父母表示愿意付费 | 商业假设初步验证 |

## 九、开放问题（待验证）

1. **3-5 岁能否独立完成涂色？** flood fill 点击填色是否足够简单？还是需要更粗放的区域？
2. **故事生成时机**：选完全部场景再生成 vs 每页选完即时生成？前者等得久，后者可能不一致
3. **线稿来源**：AI 生成 + 后处理 vs 预制线稿库？MVP 先用预制更可控
4. **6 页是否合适**：3-5 岁注意力能支撑 6 页吗？可能需要 4 页起步
